04 自律開発ループ — Cron連動の自律実装システム

Claude Codeをスケジュール実行してIssueを自動実装する仕組み

自律開発ループとは

自律開発ループは、Claude CodeをCronで定期起動し、GitHubのIssueを自動的に実装・テスト・コミットする仕組みです。

Cron (例: 毎時0分)
    ↓
Claude Code 起動（バックグラウンド）
    ↓
バックログから未実装Issueを選択
    ↓
実装 → テスト → コミット
    ↓
完了通知 → 終了

前提条件

  1. Claude Codeがヘッドレス実行できる環境
  2. GitHubのIssueにラベル管理されたバックログ
  3. テストスイートが整備されている
  4. CLAUDE.mdに自律実行時のルールが記載されている

CronCreate設定

Claude Code内から直接Cron設定が可能（durable:falseで一時的に設定）：

/loop 60m "バックログから未実装Issueを1件選んで実装してください"

または settings.json の CronCreate hook で永続設定：

{
  "crons": [{
    "schedule": "0 * * * *",
    "command": "claude --headless 'gh issue list --label backlog | head -1 | xargs claude --headless'",
    "name": "autonomous-dev-loop"
  }]
}

バックログ管理

GitHubラベル設計

ラベル 意味 自律実装
backlog 未着手 ✅ 対象
in-progress 作業中 ❌ スキップ
blocked 外部依存で止まっている ❌ スキップ
tier-1 認証・決済等の重要変更 ❌ 要人間確認

Issueの書き方（自律実装対応）

自律実装されやすいIssueの書き方：

## タスク
`src/utils/validate.py` の `validate_email()` 関数にRFCチェックを追加する

## 完了条件
- [ ] RFC 5322に準拠したメール形式をバリデーション
- [ ] 既存テスト `test_validate.py` がすべてパス
- [ ] 新しいテストケースを3件以上追加

## 参考
- 現在のコード: src/utils/validate.py:42
- テスト: tests/test_validate.py

ポイント:

  • 変更ファイルとその行番号を明示
  • 完了条件をチェックボックスで記載
  • テストへの参照を含める

CLAUDE.mdへの自律開発ルール追加

## 自律開発ループ設定

### 実装対象
- `backlog` ラベルのIssueのみ
- `tier-1` ラベルは絶対に自律実装しない

### 実装方針
- 1セッション1Issue（複数実装しない）
- テストが通らなければコミットしない
- 実装内容が曖昧な場合はIssueにコメントして終了

### コミット規則
- メッセージ: `feat: <Issue番号> <概要>`
- 完了時: Issueをクローズ

SSOT自動記録（git hook連動）

コミット時にSSOTの記録を自動で促す仕組み：

#!/bin/bash
# .git/hooks/post-commit

# コミットメッセージを取得
MSG=$(git log -1 --pretty=%B)

# Claude Codeに向けてシグナルを出力
echo "[SSOT-RECORD-TRIGGER]"
echo "Project: $(basename $(git rev-parse --show-toplevel))"
echo "Commit: $MSG"

CLAUDE.md での設定：

## SSOT自動記録（git commit連動）
Bashツール実行結果に `[SSOT-RECORD-TRIGGER]` が含まれていたら、
record-decision スキルを自動実行する。

完了通知システム

長時間タスク終了時に通知を送る：

Windowsトースト通知（WSL2環境）

#!/bin/bash
# notify-done.sh

MESSAGE="${1:-タスクが完了しました}"

# PowerShell経由でトースト通知
powershell.exe -Command "
[Windows.UI.Notifications.ToastNotificationManager, Windows.UI.Notifications, ContentType = WindowsRuntime] | Out-Null
\$template = [Windows.UI.Notifications.ToastTemplateType]::ToastText01
\$xml = [Windows.UI.Notifications.ToastNotificationManager]::GetTemplateContent(\$template)
\$xml.GetElementsByTagName('text')[0].AppendChild(\$xml.CreateTextNode('$MESSAGE')) | Out-Null
\$toast = [Windows.UI.Notifications.ToastNotification]::new(\$xml)
[Windows.UI.Notifications.ToastNotificationManager]::CreateToastNotifier('Claude Code').Show(\$toast)
"

Stop Hookと組み合わせる

{
  "hooks": {
    "Stop": [{
      "type": "command",
      "command": "~/.claude/scripts/notify-done.sh '作業が完了しました'"
    }]
  }
}

自律ループの安全ガード

自律実行中の暴走を防ぐための設定：

  1. Tier 1キーワード検知: 認証・決済・本番DB操作は自動停止
  2. テスト失敗時の停止: テストが通らなければコミットしない
  3. 実行時間上限: タイムアウト設定（例: 30分）
  4. 変更ファイル数制限: 1セッションで変更するファイルは最大10件まで
## CLAUDE.md の安全ガード設定
- Tier 1 キーワード検知時: 即停止・Issueにコメント
- テスト失敗: コミットせずに停止
- タイムアウト: 30分でセッション終了
